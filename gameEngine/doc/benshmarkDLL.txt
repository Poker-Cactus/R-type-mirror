| Caractéristique | Bibliothèque Statique (Engine.a / .lib)                                                                |
|-----------------|--------------------------------------------------------------------------------------------------------|
| Liaison         | Statique : Code moteur copié dans l'exécutable du jeu à la compilation.                                |
| Taille Exe      | Plus grand (contient tout le code moteur nécessaire).                                                  |
| Déploiement     | Très Simple : Un seul exécutable à distribuer.                                                         |
| Mise à Jour     | Complexe : Le jeu doit être recompilé en cas de changement moteur.                                     |
| Modularité      | Faible : Peu adapté aux systèmes de plugins externes.                                                  |
| Performance     | Meilleure : Liaison directe, meilleures opportunités d'optimisation par le compilateur.                |

| Caractéristique | Bibliothèque Dynamique (Engine.so / .dll)                                   |
|-----------------|-----------------------------------------------------------------------------|
| Liaison         | Dynamique : Code moteur chargé en mémoire à l'exécution (runtime).          |
| Taille Exe      | Plus petit (contient seulement les références vers le .so/.dll).            |
| Déploiement     | Plus Complexe : Nécessite l'Exécutable ET le fichier .so/.dll.              |
| Mise à Jour     | Facile : Remplacement du .so/.dll sans recompiler le jeu.                   |
| Modularité      | Élevée : Idéal pour les systèmes de plugins ou le Hot Reload.               |
| Performance     | Légèrement Inférieure : Sauts indirects (PIC), temps de chargement initial. |

Comme ces deux méthodes ont leurs avantages et inconvénients,
le choix dépendra des besoins spécifiques du projet de jeu,
tels que la facilité de mise à jour, les exigences de performance, et la complexité du déploiement.

C'est pourquoi nous utilisaeront les deux méthodes pour le cas de notre Game Engine.
La bibliothèque statique sera utilisée pour les composants critiques en termes de performance, comme le core du moteur de rendu et la physique.
La bibliothèque dynamique sera utilisée pour les modules moins critiques, comme les plugins d'extensions et les outils de développement.
